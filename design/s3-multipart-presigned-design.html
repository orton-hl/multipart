Summary

Design and document a multipart presigned-S3 upload pattern for large file uploads from untrusted clients (VDI / external partners), covering presigned URL generation, assembly, validation, retries, security, and Terraform/IAM patterns. The design must include how the backend issues presigned parts, completes the multipart upload, verifies integrity, and enforces access controls and lifecycle/purge policies.

Acceptance Criteria

  • AC-1Architecture + Sequence Diagrams — A single design doc detailing: presign flow, upload flow, complete flow, error/retry handling, and cleanup of abandoned multipart uploads.
  • AC-2Security Model — Minimum IAM permissions, short TTL for presigned URLs, optional per-upload encryption (SSE-S3 or SSE-KMS), restrict uploads to intended bucket/prefix.
  • AC-3API Contract — Specified for presign endpoints: POST /uploads → returns uploadId + presigned URLs for parts; POST /uploads/complete → client provides ETags.
  • AC-4Operational Controls — Monitoring metrics, CloudWatch/Datadog events for aborted uploads, and a cleanup job for stale multipart uploads.
  • AC-5Example Terraform snippet + Client snippet — Node or Python that demonstrates generating presigned URLs and completing the upload.
  • AC-6Security Review Checklist — Guidance for preventing malicious uploads (size limits, allowed content-types, virus scanning hook).
Definition of Done: Design doc approved, Terraform examples available in repo, PoC successfully uploads a >1GB file via multipart presigned flow and is cleaned up if abandoned.

Upload Flow Phases

Init Client calls POST /uploads. Backend creates multipart upload via SDK, generates presigned URLs per part. Returns uploadId + array of presigned PUT URLs.
Upload Client splits file into parts (≥5MB each, except last). Uploads each part via PUT to the presigned URL. Collects ETag response header per part. Supports parallel/concurrent uploads.
Complete Client calls POST /uploads/complete with uploadId and ordered ETag array. Backend calls CompleteMultipartUpload, verifies checksum, triggers post-processing.
Error / Abort On failure or timeout, backend calls AbortMultipartUpload. EventBridge cron runs daily cleanup for uploads >24h old. S3 lifecycle rule as safety net.

Part Retry Strategy

When a part PUT fails (network error, timeout, 5xx), the client retries using the same presigned URL if still within TTL, or requests a fresh URL via GET /uploads/{uploadId}/part/{partNumber}/refresh.

AttemptDelayAction
1st retry1sRetry same presigned URL
2nd retry3sRetry same presigned URL
3rd retry10sRequest fresh presigned URL, then retry
4th retry30sRequest fresh presigned URL, then retry
Fail—Surface error to user; upload remains resumable for 24h

Abort & Cleanup Flow

Explicit Abort Client calls DELETE /uploads/{uploadId}. Backend immediately calls AbortMultipartUpload. Emits upload.aborted event to CloudWatch.
Timeout Cleanup EventBridge cron (daily 03:00 UTC) invokes Cleanup Lambda. Lambda lists all in-progress multipart uploads >24h old via ListMultipartUploads. For each: calls AbortMultipartUpload and logs to CW + Datadog.
S3 Lifecycle Net S3 AbortIncompleteMultipartUpload lifecycle rule fires after 1 day as a belt-and-suspenders safety net in case Lambda is disrupted.

GET /uploads/{uploadId}/status Check upload status
Returns current status: PENDING | IN_PROGRESS | COMPLETING | COMPLETED | FAILED | ABORTED, parts uploaded so far, and expiry time.
DELETE /uploads/{uploadId} Abort upload
Aborts multipart upload and removes all uploaded parts from S3. Returns 204 No Content.

Security Review Checklist

  • 🛡️
    File size limits enforcedValidate fileSizeBytes at initiation. Max 10GB default (configurable). Reject overage before creating multipart upload. S3 bucket policy enforces s3:content-length-range condition on PUT.
  • 📋
    Content-type allowlistOnly permit declared contentType values from an allowlist (e.g., application/zip, application/octet-stream, text/csv). Reject unsupported types at API before generating presigned URLs. Include content-type in presigned URL signature.
  • 🦠
    Virus scanning hook (post-upload)On CompleteMultipartUpload, emit an S3 event notification to invoke a scan Lambda (ClamAV or commercial AV SDK). Object moves from multipart/quarantine/ to multipart/clean/ only after passing. Infected files are deleted and an alert is raised.
  • 🔐
    Encryption at restSSE-S3 default for all uploads. SSE-KMS available per-upload via "encryption": "SSE-KMS" in request. Bucket policy denies s3:PutObject unless s3:x-amz-server-side-encryption is set. KMS key rotation enabled (annual).
  • 🔑
    Authentication & authorizationAll API endpoints require JWT or API Key. Upload token scoped to partner ID and prefix. Presigned URLs are not shareable beyond intended recipient (short TTL + IP binding optional for high-risk flows).
  • 🚫
    Path traversal preventionFilename sanitized before building S3 key. Key construction uses UUIDs + server-assigned paths. No user-controlled input in S3 key path beyond the final filename segment, which is sanitized.
  • 📝
    Audit trailEvery upload initiation, completion, and abort is logged to CloudWatch Logs with uploadId, partnerId, userAgent, IP address, and timestamp. S3 server access logging enabled.
  • 🔒
    Bucket hardeningBlock all public access. No bucket ACLs. S3 Object Ownership = BucketOwnerEnforced. Deny HTTP (require HTTPS). Enforce TLS 1.2+ via bucket policy condition aws:SecureTransport.
  • 🌐
    Network controlsAPI sits behind WAF with rate limiting (100 req/min per IP). For internal VDI clients: restrict S3 access to VPC endpoint. For external partners: IP allowlisting where feasible.

resource "aws_s3_bucket_lifecycle_configuration" "uploads" {
  bucket = aws_s3_bucket.uploads.id
  rule {
    id     = "abort-incomplete-multipart"
    status = "Enabled"
    filter { prefix = "multipart/" }
    abort_incomplete_multipart_upload { days_after_initiation = 1 }
  }
  rule {
    id     = "transition-to-glacier"
    status = "Enabled"
    filter { prefix = "multipart/clean/" }
    transition { days = 90; storage_class = "GLACIER" }
  }

resource "aws_s3_bucket_policy" "uploads" {
  bucket = aws_s3_bucket.uploads.id
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        # Enforce TLS
        Sid = "DenyHTTP", Effect = "Deny"
        Principal = "*"
        Action = "s3:*"
        Resource = ["${aws_s3_bucket.uploads.arn}", "${aws_s3_bucket.uploads.arn}/*"]
        Condition = { Bool = { "aws:SecureTransport" = "false" } }
      },
      {
        # Enforce encryption on PUT
        Sid = "DenyUnencryptedPut", Effect = "Deny"
        Principal = "*"
        Action = "s3:PutObject"
        Resource = "${aws_s3_bucket.uploads.arn}/*"
        Condition = { StringNotLike = {
          "s3:x-amz-server-side-encryption" = ["AES256", "aws:kms"]
        }}
      },
      {
        # Restrict writes to multipart/ prefix only
        Sid = "RestrictPrefix", Effect = "Deny"
        Principal = { AWS = aws_iam_role.upload_svc.arn }
        Action = "s3:PutObject"
        NotResource = "${aws_s3_bucket.uploads.arn}/multipart/*"
      }
    ]
  })
}

# Cleanup Lambda Python handler (simplified)
"""
import boto3, os, logging
from datetime import datetime, timezone, timedelta

def main(event, context):
    s3 = boto3.client('s3')
    bucket = os.environ['BUCKET_NAME']
    max_age = timedelta(hours=int(os.environ.get('MAX_AGE_HOURS', 24)))
    now = datetime.now(timezone.utc)
    
    paginator = s3.get_paginator('list_multipart_uploads')
    for page in paginator.paginate(Bucket=bucket, Prefix='multipart/'):
        for upload in page.get('Uploads', []):
            age = now - upload['Initiated']
            if age > max_age:
                s3.abort_multipart_upload(
                    Bucket=bucket,
                    Key=upload['Key'],
                    UploadId=upload['UploadId']
                )
                logging.info(f"Aborted stale upload {upload['UploadId']}")
"""
// Node.js client — multipart presigned upload

import axios from 'axios';
import { createReadStream, statSync } from 'fs';

const PART_SIZE = 5 * 1024 * 1024; // 5MB
const API_BASE = 'https://api.example.com';

async function uploadFile(filePath, token) {
  const fileSize = statSync(filePath).size;
  const partCount = Math.ceil(fileSize / PART_SIZE);

  // 1. Initiate upload
  const { data: session } = await axios.post(`${API_BASE}/uploads`, {
    filename: path.basename(filePath),
    contentType: 'application/octet-stream',
    fileSizeBytes: fileSize,
    partCount,
    encryption: 'SSE-KMS',
  }, { headers: { Authorization: `Bearer ${token}` } });

  const { uploadId, presignedUrls } = session;
  const parts = [];

  // 2. Upload parts (with retry)
  await Promise.all(presignedUrls.map(async ({ partNumber, url }) => {
    const start = (partNumber - 1) * PART_SIZE;
    const end = Math.min(start + PART_SIZE, fileSize);
    const chunk = readChunk(filePath, start, end);

    const res = await withRetry(() => axios.put(url, chunk, {
      headers: {
        'Content-Type': 'application/octet-stream',
        'Content-Length': chunk.length,
      }
    }));
    parts.push({ partNumber, etag: res.headers['etag'] });
  }));

  // 3. Complete upload
  const { data: result } = await axios.post(`${API_BASE}/uploads/complete`, {
    uploadId,
    parts: parts.sort((a, b) => a.partNumber - b.partNumber),
  }, { headers: { Authorization: `Bearer ${token}` } });

  return result; // { fileKey, etag, sizeBytes, status }
}

async function withRetry(fn, maxAttempts = 4) {
  const delays = [1000, 3000, 10000, 30000];
  for (let i = 0; i < maxAttempts; i++) {
    try { return await fn(); }
    catch (err) {
      if (i === maxAttempts - 1) throw err;
      await new Promise(r => setTimeout(r, delays[i]));
    }
  }
}

Capacity Planning Note: At 5MB parts and 10K max parts per upload, the theoretical max object size is ~50GB. Default API limit is 10GB. For files over 50GB, consider chunking or using S3 Transfer Acceleration. Each concurrent upload holds temporary storage in S3 until completed or aborted — monitor bucket size during high-volume periods.
